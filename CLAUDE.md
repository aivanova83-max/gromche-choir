# CLAUDE.md — Хор «Громче»

## 1. Project Description

Публичный сайт женского хора «Громче» (Москва и МО) — лендинг/витрина с информацией о хоре, расписанием занятий и формой записи. Целевая аудитория: женщины, которые хотят петь в хоре без профессиональных требований.

## 4. Conventions

### Коммиты
Проект создан через Lovable — коммиты от Lovable идут в свободном формате. Для ручных правок:
```
feat: добавить секцию расписания
fix: исправить ширину карточки на мобиле
style: выровнять отступы в Hero
chore: обновить зависимости
```
- Язык: русский
- Тело коммита: опционально, описывает «почему», а не «что»

### Именование файлов и компонентов
- Компоненты: `PascalCase.tsx` → `HeroSection.tsx`, `ScheduleCard.tsx`
- Хуки: `camelCase` с префиксом `use` → `useSchedule.ts`
- Утилиты/хелперы: `camelCase.ts`
- Страницы: `PascalCase.tsx` в `src/pages/`
- CSS-переменные: `--kebab-case` (уже заведены в tailwind.config.ts)

### Ветки
- Основная: `main`
- Разработка: `claude/init-session-pEhXe` (текущая сессия)
- Паттерн для фич: `feat/<название>` или `fix/<название>`

### Стили
- Использовать **только Tailwind-классы** и кастомные CSS-переменные из `tailwind.config.ts`
- Кастомная палитра: `olive`, `warm-cream`, `warm-peach`, `warm-text`, `warm-bg`
- Кастомные градиенты: `bg-gradient-warm`, `bg-gradient-hero`
- Кастомные тени: `shadow-soft`, `shadow-warm`
- Кастомные анимации: `animate-fade-in`, `animate-float`, `animate-pulse-subtle`
- Dark mode через `class` (конфиг уже есть) — [заполни сам: используется ли тёмная тема на сайте?]

## 6. What Claude Should Know

### Бизнес-контекст
- Сайт для **женского хора** без профессионального ценза — тон тёплый, поддерживающий, не академический
- Слоган: «Ты можешь громче!»
- Главный CTA — **запись на занятие** через виджет MoyKlass (уже вставлен в `index.html` как `<div id="SiteWidgetMoyklass133029">`)
- Аудитория — женщины любого возраста, не обязательно с музыкальным образованием

### Технические особенности
- **Lovable-платформа**: в dev-режиме работает `lovable-tagger` — добавляет `data-component-id` атрибуты к компонентам для визуального редактора. Это нормально, не удалять
- **TypeScript non-strict**: проект намеренно использует мягкие TS-настройки (совместимость с Lovable-генерацией). Не ужесточать без необходимости
- **shadcn/ui**: компоненты в `src/components/ui/` генерируются через CLI (`npx shadcn@latest add ...`). Редактировать их напрямую нежелательно — изменения могут перезатереться
- **Dev-порт**: 8080 (не стандартный 5173)
- **MoyKlass**: внешний виджет записи на занятия. Логика записи — на стороне MoyKlass, не дублировать её в коде

### Дизайн-система
- Вся цветовая схема через CSS-переменные — цвета в `tailwind.config.ts` ссылаются на `hsl(var(--...))`. Значения переменных должны быть в `src/index.css`
- [заполни сам] Опиши основные цвета бренда (hex/hsl) когда будет готов дизайн

### Страницы / маршруты
| Маршрут | Файл | Описание |
|---|---|---|
| `/` | `pages/Index.tsx` | Главная страница (лендинг) |
| `/events` | `pages/Events.tsx` | Афиша мероприятий |
| `/events/mother` | `pages/EventMother.tsx` | Страница прошедшего мероприятия |
| `/zapis` | `pages/Zapis.tsx` | Запись на занятие — виджет MoyKlass |
| `/legal` | `pages/Legal.tsx` | Перечень юридических документов |

> **Замечание:** страница `/events/mother` — пример страницы конкретного мероприятия. По мере появления новых событий паттерн именования: `/events/<slug>`.

## 7. Don't

1. **Не трогать `src/components/ui/`** — это shadcn-компоненты. Кастомизация только через CSS-переменные в `index.css` или обёртки поверх них.
2. **Не убирать `lovable-tagger`** из `vite.config.ts` и не удалять `data-component-id` атрибуты из JSX — они нужны для интеграции с платформой Lovable.
3. **Не менять порт** dev-сервера с 8080 — Lovable ожидает именно этот порт для превью.
4. **Не ужесточать TypeScript** (`strict: true`, `noImplicitAny: true`) без согласования — это сломает совместимость с кодом, генерируемым Lovable.
5. **Не хардкодить тексты и цвета** напрямую в компонентах — использовать CSS-переменные для цветов и константы/переводы для текста.
6. **Не коммитить `.env`** файлы — чувствительные данные только в переменных окружения, в репозитории только `.env.example`.
